# Engineering Student Placement Resume Review Rubric

This document combines the core 100-point evaluation rubric and the detailed formatting, structure, and keyword guidelines extracted from the placement office guidelines (`Resume Review Doc.pdf`).

---

## Part 1: Core Scoring Rubric (100 Points Total)

Resumes must be evaluated strictly based on content, not layout (as the placement platform Superset enforces layout). However, the resume **must be strictly one page**.

### 1. Structure/Field Completeness (10 Points)
* **Criteria**: All relevant Superset fields filled meaningfully.
* **Deductions**: No "N/A," blank sections, or leftover placeholder text where real content should be.

### 2. Content Quality — Action Verbs & Quantification (30 Points)
* **Strong Action Verbs (10 pts)**: Bullets must start with strong action verbs (e.g., *Improved, Collaborated, Analyzed*). Penalize passive phrases like "responsible for," "helped," "worked on."
* **Quantification of Achievements (10 pts)**: Achievements must be quantified using percentages, currency symbols (₹/$), time, scale, or metrics rather than vague claims.
* **Implicit STAR Logic (10 pts)**: Content should follow the STAR framework (Situation, Task, Action, Result) presenting context, action, and a measurable result (even if not explicitly labeled).

### 3. Projects & Experience Depth (25 Points)
* **Technical/Functional Depth (10 pts)**: Demonstrate real technical/functional depth instead of a surface-level listing of technologies.
* **Relevance to Target Roles (10 pts)**: High relevance to electronics, IT-adjacent, or general engineering roles.
* **Progression/Ownership (5 pts)**: Internships/Positions of Responsibility (PORs) should show career/academic progression or ownership rather than simple attendance.

### 4. Skills & Academic Fit (10 Points)
* **Skills Relevance**: Specific and relevant skills rather than generic keyword-stuffing.
* **Academic Representation**: CGPA and coursework represented appropriately.

### 5. Overall Human Read — Gut Check (25 Points)
* **Sanity Check**: If a recruiter skims this resume for 20 seconds, would it land well?
* **Polished & Confident (12 pts)**: Feels polished, confident, and ready to send as-is.
* **Coherent Story (13 pts)**: Tells a coherent story about the candidate, rather than reading like a checklist was filled in mechanically.
* *Note*: This section is deliberately holistic. If the mechanical score from sections 1-4 doesn't match how the resume reads overall, use this section to adjust the total score up or down by a few points. Do not penalize the same minor flaw twice.

---

## Part 2: Red Flags (Deduct 5 Points Each, Stacking, No Cap)
* **Unprofessional Email Address**: E.g., `coolboy123@gmail.com` or other informal handles.
* **Unexplained Timeline Gaps**: Significant unaccounted periods between academic/employment events.
* **Factual Inconsistencies**: Overlapping dates, inconsistent grade representations, etc.
* **Empty Core Section**: A crucial section (e.g., Projects, Internships) left empty.
* **Placeholder/Template Text**: Leftover instructions, dummy content, or prompt-like phrases.
* **Overcrowded Content**: Visually cramped layout to force-fit the one-page limit.

---

## Part 3: Color Mapping & Thresholds
Based on the final total score (after all deductions):
* **🟢 GREEN (80–100)**: Excellent/Good resume. Ready to apply or meets most rubric criteria.
* **🟡 YELLOW (55–79)**: Average resume. Needs improvement on quantified bullet points, project details, or structural elements.
* **🔴 RED (Below 55)**: Poor resume. Fails key criteria, contains severe red flags, or missing major sections.

---

## Part 4: Strict Remark Writing Rules
The AI remarks must adhere strictly to these rules:
1. **Length**: 2–3 sentences, under ~50 words total.
2. **Specificity**: Reference something specific from the resume's text (e.g., project name, particular skill, missing section). Never use generic praise or criticism.
3. **Structure**:
   * **Sentence 1**: What's working (if anything is genuinely strong).
   * **Sentence 2**: The single biggest gap (explicitly naming which rubric category it falls under).
   * **Sentence 3**: One concrete, actionable fix.
4. **Constraints**:
   * **Do NOT** restate the score or color inside the remark.
   * **If Red Flags** are triggered, explicitly name which one(s).

## Part 5: Superset & Resume Review Doc PDF Guidelines

These guidelines represent best practices extracted from the placement documentation (`Resume Review Doc.pdf`) for student resumes.

### A. Standard Resume Structure

| Section | Details & Requirements |
| :--- | :--- |
| **Header** | **Personal Details**: Name, Image (passport size), College, Course.<br>**Contact Details**: Email (college + personal), Phone, Linkedin, GitHub, Portfolio Link. |
| **Academic Details** | In reverse chronological order: Post Grad, Grad, Class XII, Class X. School names should be single-lined/compressed. Center-aligned text is preferred. |
| **Skills & Interests** | **Interests**: Broad but specific (e.g., Finance, Product, Chess, Data Science).<br>**Tools/Tech**: Python, SQL, Jira, Figma, Tableau, etc. (No keyword stuffing). |
| **Work Ex / Internships** | 3-4 Relevant & recent roles. Must include single-line company descriptions and product-like titles if SDE/UIUX/Analyst. |
| **Projects** | 1. **Dev/Design Based** (SDE, UI/UX)<br>2. **Data/Business Based** (AI/ML, SQL/Excel, Case Studies)<br>3. **Product Based** (Teardowns & Case Studies) |
| **PORs** | 1-3 Positions of Responsibility to show managerial, communication, and leadership skills. |
| **Awards & Achievements** | Hackathons, sports tournaments, publications, institute-level awards. |

### B. Formatting & Style Guidelines
* **Strict Page Limit**: Maximum 1 page.
* **Column Layout**: A single column is preferred to ensure ATS scanner compatibility.
* **Chronological Order**: Always list entries within sections in reverse chronological order.
* **Font Sizes**: Relative font sizes must be close (avoid massive headers with tiny body text).
* **White Space**: Maintain standard spacing. Excessive whitespace implies lack of experience; too little indicates overcrowding.

### C. Role Title Transformations
If students held generic developer or operations roles, they should use "Product-Like" titles to highlight their focus, keeping at least one experience with their actual title if listing 3 or more experiences:

| Actual Title | Suggested Title in Resume | Explanation standard |
| :--- | :--- | :--- |
| Software / FE / BE Developer | **Product Developer** | Focus on features, metrics, and user-stories |
| UI / UX Designer | **Product Designer** | Focus on wireframes, user flows, and Figma |
| Data / Business Analyst | **Product Analyst** | Focus on datasets, metrics, and user retention |
| Community Manager | **Growth Manager** | Focus on acquisition channels and users |
| Social Media / SEO Manager | **Product Marketing Manager** | Focus on user acquisition and metrics |
| Operations Manager | **Generalist** | Focus on project execution and stakeholder alignment |

### D. Single-Line Company Description Samples
Provide context on the company size, line of business, and funding:
1. *"Hiring, Navigation & Community App for Truckers"*
2. *"Consumer & Enterprise SaaS Solutions for Salon Industry"*
3. *"AI-based Ed-Tech startup focused on Kindergarten Kids"*

### E. Bullet Point Contrast (Incorrect vs. Correct)
* **❌ Incorrect (Vague & Passive)**:
  * *Web Developer, Website making using React, Eftax Co. [Dec-Feb 2020-21]*
  * Created website using React.js.
  * Learned JavaScript, React and Ant UI.
  * Made a fully functional web app in two months which is a potential business.
* **✅ Correct (Action Verbs, Quantification, STAR)**:
  * *Product Developer - Sugee, Japan (Apr '20 - May '20)*
  * Studied **5 different User Personas** to chalk out requirements for a 'LinkedIn for Founders' Web App.
  * Made a fully functional web app using **React.js, MongoDB & Python** in 2 months to cater to **10K users**.
  * Learned Japanese business manners, **JavaScript, Figma and Ant UI on-the-go** required for the project.

### F. Recommended Keywords by Domain
* **Strategy**: *Product vision, roadmap, competitor analysis, user persona, strategic review.*
* **UX/UI Design**: *Wireframes, user feedback, design system, competitor UX, drop-off points.*
* **Coding**: *User stories, technical trade-offs, frontend/backend integration, multithreaded.*
* **Digital Marketing**: *Acquisition channels, value propositions, user retention, DAU/MAU.*
* **Data Analysis**: *Key success metrics, large datasets, actionable insights, retention, SQL/Tableau.*
* **Leadership**: *Stakeholder alignment, managed project, led initiative, cross-functional.*
